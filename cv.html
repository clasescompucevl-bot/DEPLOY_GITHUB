Resumen Completo de HTML

HTML (HyperText Markup Language) es el lenguaje que define la estructura de una página web.
No es un lenguaje de programación.

HTML5 esencial para proyectos reales (2025)

1) Estructura mínima:

<!DOCTYPE html> <!-- Indica HTML5 -->
<html lang="es"> <!-- Idioma obligatorio -->
<head> <!-- Información de la página (no se ve en pantalla) -->
  <meta charset="UTF-8"> <!-- Codificación universal -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0"> <!-- Responsive -->
  <title>Mi Proyecto</title> <!-- Título que aparece en la pestaña del navegador -->
  <link rel="icon" href="/favicon.ico" type="image/x-icon">
  <!-- Estilos -->
  <link rel="stylesheet" href="estilos.css">  <!-- Para poner archivos de estilos -->
</head>
<body> <!-- Todo el contenido visible: textos, imágenes, botones -->
</body>
</html>

Nueva Pestaña
Contenido visible en la pestaña "Mi Proyecto"

En este caso el contenido visible es: el favicon y el title

Elemento Descripción
<!DOCTYPE html> Indica al navegador que el documento usa HTML5
<html lang="es"> Define el idioma de la página
<head> Contiene metadatos y enlaces, no se ve directamente en pantalla
<meta charset="UTF-8"> Codificación universal para que los caracteres se muestren correctamente
<meta name="viewport" content="width=device-width, initial-scale=1.0"> Hace que la página sea responsive (adaptable a dispositivos móviles)
<title> Título que aparece en la pestaña del navegador
<link rel="icon" href="/favicon.ico" type="image/x-icon"> Favicon visible en la pestaña
<body> Contenido visible de la página: texto, imágenes, botones, etc.
<link rel="stylesheet" href="estilos.css"> Para cargar estilos
Continuación: HTML moderno y semántico

HTML5 introdujo etiquetas que hacen el código más legible y mejor para SEO y accesibilidad.

Estructura semántica

En vez de usar <div> para todo, ahora existen:

  <header>   <!-- Encabezado de la página -->
  <nav>      <!-- Barra de navegación -->
  <main>     <!-- Contenido principal -->
  <section>  <!-- Sección temática -->
  <article>  <!-- Artículo independiente -->
  <aside>    <!-- Contenido lateral -->
  <footer>   <!-- Pie de página -->

Buenas prácticas básicas: Usar etiquetas semánticas cuando se pueda

Nota: Aside dentro del <main> → contenido relacionado con la sección principal. Aside fuera del <main> → contenido no relacionado, como información general o complementaria de la página, es recomendable dejarlo fuera del main.

Ejemplo 3: Puede haber sections dentro de article

Cada artículo puede dividir su contenido en secciones internas. Por ejemplo, una noticia puede tener una sección de resumen, otra de desarrollo y otra de comentarios. Esto ayuda a organizar mejor la información y mantener la semántica correcta. 

        <!DOCTYPE html>
        <html lang="es">
        <head>
        <meta charset="UTF-8">
        <title>Ejemplo 4 - Sections dentro de Article</title>
        </head>
        <body>
        <main>
          <article>
            <h2>Noticia: Lanzamiento de Tecnología</h2>
            <section>
              <h3>Resumen</h3>
              <p>Se presenta la nueva línea de productos tecnológicos con innovaciones importantes...</p>
            </section>
            <section>
              <h3>Detalles Técnicos</h3>
              <p>El producto incluye características como IA integrada, batería de larga duración y pantalla flexible...</p>
            </section>
            <section>
              <h3>Comentarios de Usuarios</h3>
              <p>Los primeros usuarios destacan su facilidad de uso y rendimiento...</p>
            </section>
          </article>
        </main>
        </body>
        </html>

Ejemplo 4: enunciado y cómo pensarlo:

Enunciado:
Diseña una plataforma digital integral que combine noticias internacionales, educación online y e-commerce de tecnología.

La página debe incluir:
Header con el logo, nombre del sitio y menú principal.

Noticias destacadas en portada, con secciones de “Política”, “Economía” y “Deportes”, y cada noticia con título, subtítulo, imagen, cuerpo y autor.

Bloque de opinión y editorial al final de la sección de noticias.

Página educativa de cursos profesionales, con categorías (Programación, Diseño, Marketing). Cada curso debe tener descripción, objetivos, listado de clases, profesor, y cada clase con video, material descargable y comentarios de alumnos.

Portal de e-commerce que muestre productos destacados, ofertas y colecciones. Cada producto tiene nombre, precio, descripción, imagen y reseñas. Además, debe haber una sección de noticias sobre lanzamientos y un blog con guías y tutoriales.

Barra lateral para cada sección, mostrando artículos relacionados, publicidad, recomendaciones, filtros de productos o cursos relacionados.

Footer con contacto, redes sociales, métodos de pago, derechos y políticas del sitio.

Elementos de Texto en HTML

Títulos

<h1>: Un solo título principal por página. Ejemplo: nombre de la web o título del artículo.

<h2>: Subtítulos dentro del tema principal.

<h3>–<h6>: Subniveles según necesidad. Nunca uses <h3> si no hay un <h2> antes.

  <h1>Nombre de la Web</h1>
  <h2>Subtítulo de Sección</h2>
  <h3>Subnivel de Contenido</h3>

Texto

<p>: Párrafos de contenido normal.

<strong>: Resaltar lo realmente importante en el sentido semántico (no solo visual).

<em>: Marcar énfasis, tono diferente o palabra destacada.

  <p>Este es un párrafo normal.</p>
  <p>Este texto es <strong>muy importante</strong> y este tiene <em>énfasis</em>.</p>
Ejemplo de Texto

Este es un párrafo normal.

Este texto es muy importante y este tiene énfasis.

Texto Avanzado

<mark>: Señalar algo a resaltar como “subrayado fluorescente”.

<abbr>: Cuando uses abreviaturas o siglas. Ejemplo: CSS.

<code>: Código en línea dentro de un párrafo.

<pre>: Bloque de texto que debe conservar saltos de línea y espacios (código, ASCII art).

  <p>Resalta <mark>texto importante</mark> en un párrafo.</p>
  <p>Usa <abbr title="Cascading Style Sheets">CSS</abbr> para estilos.</p>
  <p>Ejemplo de código: <code>let x = 10;</code>.</p>
  <pre>
  function ejemplo() {
    console.log("Hola");
  }
  </pre>
Ejemplo de Texto Avanzado

Resalta texto importante en un párrafo.

Usa CSS para estilos.

Ejemplo de código: let x = 10;.

  function ejemplo() {
    console.log("Hola");
  }

Citas

<blockquote>: Citas largas de otra fuente, en bloque.

<q>: Cita corta dentro de una frase.

  <blockquote>
    Esta es una cita larga de una fuente externa.
  </blockquote>
  <p>Ella dijo <q>Esto es una cita corta</q> dentro de un párrafo.</p>

Misceláneo

<br>: Solo para saltos de línea donde no corresponde abrir un nuevo párrafo (direcciones, poesía, etc.).

  <p>
    Dirección:<br>
    Calle Principal 123<br>
    Ciudad, País
  </p>

Elemento Descripción
<h1> Título principal, solo uno por página
<h2> Subtítulos dentro del tema principal
<h3>–<h6> Subniveles, respetar jerarquía
<p> Párrafos de contenido normal
<strong> Resaltar importancia semántica
<em> Marcar énfasis o tono diferente
<mark> Resaltar como “subrayado fluorescente”
<abbr> Abreviaturas o siglas con descripción
<code> Código en línea dentro de un párrafo
<pre> Bloque de texto con formato preservado
<blockquote> Citas largas en bloque
<q> Citas cortas dentro de una frase
<br> Salto de línea para casos específicos
Enlaces en HTML

Enlaces

Los enlaces en HTML se crean con la etiqueta <a>, permitiendo navegar a otras páginas, recursos o archivos HTML.

    <a href="https://www.google.com" target="_blank" rel="noopener noreferrer">Ir a Google</a>
    <a href="#contacto">Ir a Contacto</a>
    <a href="about.html">Ir a Acerca</a>
    <h2 id="contacto">Sección de Contacto</h2>

Explicación de Atributos

Atributo Descripción
href Dirección del enlace (URL externa, #id para enlaces internos, o ruta a otro archivo HTML como "about.html").
target="_blank" Abre el enlace en una nueva pestaña.
rel="noopener noreferrer" Obligatorio con target="_blank" por seguridad (evita acceso al origen desde la nueva ventana) y rendimiento.
#id Permite navegar a un elemento específico de la misma página que tenga ese id.
